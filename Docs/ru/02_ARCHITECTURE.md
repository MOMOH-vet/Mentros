# Архитектура
- Подход: компонентный, с разделением по модулям.
- Данные: через ScriptableObjects и конфигурационные ассеты.
- Префабы: используются как шаблоны игровых объектов.
- Ответственности модулей:
  - Contracts: интерфейсы, перечисления, идентификаторы.
  - Core: базовая инфраструктура и утилиты.
  - Input: сбор и маршрутизация ввода.
  - Content: каталоги и классы данных.
  - Tanks: логика танков.
  - Combat: боевая логика.
  - AI: поведение противников.
  - Levels: правила и сценарии уровней.
  - UI: экраны и виджеты.
  - Persistence: сохранение/загрузка.
- Правило границ: UI не зависит напрямую от Tanks/Combat/AI/Levels.
